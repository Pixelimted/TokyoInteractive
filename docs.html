Documentation

Everything you need to set up and run Unknown in your server, from the mod role to the Roblox console.

Getting Started

1

Add Unknown to your server

You'll need Manage Server permission on the server you're adding it to.

2

Run /setup

Set your mod role and log channels. Every option in this command is optional, so you can run it multiple times to change one thing at a time.

3

Log into the dashboard

For a visual view of cases, live stats, and settings, sign in at the dashboard with Discord.

The /setup command

Unknown uses one /setup command for all configuration, rather than a separate subcommand for every setting. Run it with no options to see your current configuration, or fill in only the fields you want to change.

modrole
Role that can use moderation commands
summarylog
Channel for short one-line case summaries
detailedlog
Channel for full case cards
warnkickat
Auto-kick after this many warnings (1-20)
warnbanat
Auto-ban after this many warnings (1-20)
dmonaction
Whether users get DMed when actioned
logleaves
Whether member departures get logged
prefix
Text command prefix, e.g. ! or .
aimoderation
Turn AI Moderation on or off (beta)
aimodchannel
Review channel for AI Moderation flags

Moderation commands

/ban
Ban a user, permanently or for a set duration
/unban
Lift a ban using a user ID
/kick
Remove a user from the server
/mute
Timeout a user for a given duration
/unmute
Lift an active timeout early
/warn
Log a warning, counts toward auto-escalation
/unwarn
Remove a warning by case ID
/note
Attach a private mod-only note to a user
/cases
View a user's full moderation history
/case
Look up one case by its ID
/viewlog
See recent moderation activity server-wide
/reason
Edit the reason on an existing case
/roblox
Link, view, or remove a Roblox username
/purge
Bulk delete messages, optionally by user
/slowmode
Set a channel's slowmode delay
/lock, /unlock
Restrict or restore send permissions in a channel
/userinfo
Pull account age, roles, and case count

Every command checks both your mod role and your Discord permissions. Having the mod role isn't enough on its own if Discord's own permission system would also block the action — both need to allow it.

Prefix commands

If your moderators prefer typing over slash commands, set a prefix in /setup or from the dashboard. Once set, commands like this work the same as their slash equivalents:

!ban @user reason
Same as /ban
!warn @user reason
Same as /warn
!cases @user
Same as /cases

Note: prefix commands are aliases only. They call the exact same underlying logic as the slash commands, so there is never a difference in what actually happens.

AI Moderation (beta)

When enabled, Unknown scans messages for patterns commonly associated with problem behavior: known scam and phishing link formats, slur variants, and mass-mention spam. Anything that matches gets posted to a review channel of your choice.

Important: this is pattern matching, not a model that understands context or intent. It will miss things, and it will occasionally flag something harmless. It never takes action on its own — a human moderator reviews every flag and decides what to do.

Roblox console

Unknown can run Cmdr commands directly inside your live Roblox servers. In Discord, use /ingame to pick a server from a list of everything currently online, public or private, then type your command. From the dashboard, the Roblox Console tab does the same thing, targeting whichever live server has the most players.

Using the dashboard

The dashboard shows real data pulled live from the bot: case counts, active mutes, a breakdown by action type, and the 25 most recent moderation actions. Only servers where you have Manage Server permission and where Unknown is actually present will show up.

Settings you change on the dashboard take effect immediately, the same as running /setup in Discord.
